安装Memcached

安装libevent依赖

tar zxvf libevent-2.0.22-stable.tar.gz
cd libevent-2.0.22-stable
./configure --prefix=/Data/apps/libs
make
make install

安装memcached

1
2
3
4
5

tar zxvf memcached-1.4.27.tar.gz
cd memcached-1.4.2
./configure --prefix=/Data/apps/memcached --with-libevent=/Data/apps/libs
make
make install
  • 启用 通常：启动Memcache的服务器端的命令为：
1

/Data/apps/memcached/bin/memcached -d -m 10 -u www -l 192.168.232.129 -p 11211 -c 256 -P /tmp/memcached.pid

-d选项是启动一个守护进程， -m是分配给Memcache使用的内存数量，单位是MB，我这里是10MB， -u是运行Memcache的用户，我这里是root， -l是监听的服务器IP地址，如果有多个地址的话，我这里指定了服务器的内网IP地址192.168.232.129 -p是设置Memcache监听的端口，我这里设置了11211，最好是1024以上的端口， -c选项是最大运行的并发连接数，默认是1024，我这里设置了256，按照你服务器的负载量来设定， -P是设置保存Memcache的pid文件，我这里是保存在 /tmp/memcached.pid，

想开机自动启动的话，只需在/etc/rc.d/rc.local中加入一行，上面命令 有人用以下命令：

1

/Data/apps/memcached/bin/memcached -d -m 20 -p 11211 -u www

上面有些东西可以参考一下：即，ip不指定时，默认是本机，用户，最好选择是：www 这样，也就是属于哪个用户的服务，由哪个用户启动。

# php代码
<?php
    $memcache = new Memcache; //创建一个memcache对象
    $memcache->connect('192.168.232.129', 11211) or die ("Could not connect"); //连接Memcached服务器
    $memcache->set('key', 'test'); //设置一个变量到内存中，名称是key 值是test
    $get_value = $memcache->get('key'); //从内存中取出key的值
    echo $get_value;
?>
